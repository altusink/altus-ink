# IMPLEMENTATION PROMPT v3: Feature Updates & Refinement
**Objective**: Apply specific user-requested improvements to the ALTUSINK platform. Focus on routing, pricing tiers, cover-up flow, and authentication connections.
## 1. Routing & Middleware Updates
**Goal**: Ensure the landing page is always the default entry point.
- **Modify `middleware.ts`**:
  - Remove any auto-redirect from `/` to `/dashboard`.
  - Root path `/` must always render the Landing Page.
  - Dashboard routes (`/dashboard/*`) remain protected.
## 2. Database Schema Update
**Goal**: Support new pricing tiers and "Cover-up" specific logic.
- **Update `deposit_values` table**:
  - Add `duration_hours` as `DECIMAL(4,1)` (to support 1.5h).
  - Add `service_type` enum: `'standard'`, `'coverup'`.
  - Add `cta_action` enum: `'payment'`, `'whatsapp'`.
```sql
-- Run this SQL Migration
ALTER TABLE public.deposit_values 
  ALTER COLUMN duration_hours TYPE DECIMAL(4,1),
  ADD COLUMN service_type TEXT DEFAULT 'standard' CHECK (service_type IN ('standard', 'coverup')),
  ADD COLUMN cta_action TEXT DEFAULT 'payment' CHECK (cta_action IN ('payment', 'whatsapp'));
-- Insert Default Values (for new artists)
INSERT INTO public.deposit_values (artist_id, name, duration_hours, deposit_amount, service_type, cta_action)
SELECT 
  id as artist_id,
  name, duration, amount, type, action
FROM public.artists
CROSS JOIN (VALUES
  ('Small', 1.5, 30.00, 'standard', 'payment'),
  ('Medium', 3.0, 50.00, 'standard', 'payment'),
  ('Large', 5.0, 80.00, 'standard', 'payment'),
  ('XL', 8.0, 150.00, 'standard', 'payment'),
  ('Cobertura P', 2.0, 100.00, 'coverup', 'payment'),
  ('Cobertura M', 4.0, 200.00, 'coverup', 'payment'),
  ('Cobertura G', 6.0, 300.00, 'coverup', 'payment'),
  ('Cobertura XL', 8.0, 350.00, 'coverup', 'whatsapp')
) AS v(name, duration, amount, type, action)
WHERE NOT EXISTS (SELECT 1 FROM public.deposit_values WHERE artist_id = public.artists.id);
```
## 3. Booking Flow UI Updates
**Goal**: Implement the new Pricing Table and Cover-up Menu.
- **Service Selection Step**:
  - Add a **Tab Switcher**: [Tattoo (Standard)] | [Cover-up (Reforma)]
  - **Standard Tab**:
    - Display cards: Small (1h-1:30h / €30), Medium (2h-3h / €50), Large (3-5h / €80), XL (5-8h / €150).
  - **Cover-up Tab**:
    - Display cards: P (€100), M (€200), G (€300).
    - **XL Card (€350)**: Instead of "Book", show button "Discuss on WhatsApp" (links to Artist's WhatsApp).
- **Real-time Optimization**:
  - Use `useOptimistic` for immediate UI feedback when selecting slots.
  - Prefetch availability when hovering over dates.
## 4. Authentication Connectivity
**Goal**: Ensure no dead buttons.
- **Global Rule**: Any interaction that creates data (Booking, Saving Favorite, etc.) must check session.
- **Implementation**:
  - If user is NOT logged in: Redirect to `/login?next=current_path`.
  - "Book Appointment" button -> Check Auth -> If No -> Login Modal/Redirect.
  - "Dashboard" link in Nav -> Only show if logged in.
## 5. Suggested Improvements (Proactive)
- **Cover-up Photo Upload**: Add a specific step for Cover-ups to upload a photo of the *current* tattoo to be covered.
- **Anamnesis Form**: digital health questionnaire before booking.
- **Waitlist Deposit**: Option to pay a small fee to join the priority waitlist.
